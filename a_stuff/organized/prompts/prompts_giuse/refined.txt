PLANNING PRINCIPLES

- Correctly identify the final task target.
- Identify only objects that physically block the execution of the task.
- Add clearing steps only when strictly necessary.
- Prefer the shortest valid plan.
- Do not invent objects that are not visually present.
- Use consistent object naming across the plan.
- Ensure that every step is causally justified.
- Ensure that preconditions and postconditions are consistent across steps.
- Do not include irrelevant objects in the blocking_objects list.
- If the task can be completed directly, do not add unnecessary clearing steps.

The robot has two arms: left_arm and right_arm.

Manipulation modes:

single_arm

dual_arm_symmetric  
Both arms act on the same object with the same role.

dual_arm_asymmetric  
Both arms participate in the same coordinated task but with different roles.

dual_arm_parallel  
Each arm performs a different action in parallel, typically on different objects.


FORMATTING CONSTRAINTS

- Maximum 4 steps.
- All required fields must be present.
- Use only the allowed taxonomy values.
